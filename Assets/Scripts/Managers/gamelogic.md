# FaceTheRandomness - 게임 로직 및 보조 시스템 정리

이 문서는 게임을 관리하는 전체적인 핵심 로직과 이를 뒷받침하는 보조 시스템(유틸리티)들의 역할과 동작 방식을 정리한 문서입니다.

---

## 1. 게임 관리 핵심 로직

이 프로젝트에서 게임 세션과 흐름 관리는 주로 세 가지 핵심 클래스(`GameLoader`, `GameManager`, `Game`)를 중심으로 유기적으로 작동합니다. 

### 1-1. GameLoader (초기화 및 전역 시스템 관리)
`GameLoader`는 게임이 켜질 때 가장 먼저 실행되며, 앱 전반에 걸쳐 유지되어야 하는 기반 시스템을 세팅합니다.
* **전역 시스템 초기화**: `Awake()` 단계에서 `CoroutineHelper`, `Printer`, 전역 `TimeManager` 등 싱글턴 객체들을 생성하고 목표 프레임(60fps)을 고정합니다.
* **씬 로드 관리**: `LoadScene` 및 `UnloadScene` 함수를 제공하여 비동기로 씬을 전환하는 역할을 수행합니다. 게임 시작 시 가장 먼저 타이틀 씬(`TITLE_SCENE_NAME`)을 로드하고 `TitleUI`를 띄웁니다.
* **UI 상호작용 감지**: `Update()` 문을 통해 화면 터치나 마우스 클릭을 감지합니다. 버튼 UI 요소를 누를 때마다 전역적으로 클릭 효과음(SFX)을 재생하도록 입력 처리를 담당합니다.

### 1-2. GameManager (게임 흐름 및 씬 전환 제어)
`GameManager`는 전체적인 '게임의 라이프사이클(시작, 종료, 승리 등)'을 제어하는 매니저(싱글턴)입니다.
* **게임 시작 및 데이터 로드**: 게임이 시작될 때(`StartGame`), 로드되지 않은 근거리/원거리 무기 데이터(`WeaponDefinition`)를 가져와 리스트화합니다.
* **게임 세션 관리**: 게임 씬(`GAME_SCENE_NAME`)을 로드한 직후, **`Game` 객체를 새로 생성**하여 현재 진행 중인 게임 세션(`CurrentGame`)으로 할당하고 시작시킵니다.
* **Update 루프 제어**: 자신의 `Update()` 안에서 `CurrentGame.Update()`를 호출하여 인게임 로직이 매 프레임 돌아가도록 제어합니다.
* **승리/패배 상태 처리**: `GameOver()`, `GameClear()` 함수를 통해 결과 UI와 토스트 메시지를 띄우고, 일정 딜레이 후 리소스를 정리하며 다시 타이틀 씬으로 돌아가는(`ReturnToTitle`) 흐름을 관장합니다.

### 1-3. Game (단일 게임 세션 제어)
`Game` 클래스는 유니티의 컴포넌트(MonoBehaviour)가 아닌 일반 클래스 형태를 가지며, 플레이어가 진행하는 **한 판의 인게임 세션(Session)** 안에서 벌어지는 모든 상태를 통제합니다.
* **초기화 및 생성 (`Start`)**: `Player` 생성 및 이동 스크립트 연결, `InGameUI` 띄우기, 맵 생성(`MapGenerator`) 및 적 시스템(`EnemySystem`) 연동을 수행합니다.
* **인게임 업데이트 (`Update`)**:
  * 자체 `_timeManager`를 사용해 인게임 시간을 계산합니다.
  * UI 상태 갱신, ESC 키를 통한 설정창 열기 등을 처리합니다.
  * 플레이어가 획득한 카드(`GoodCards`, `BadCards`)들의 효과를 매 프레임마다 적용합니다.
  * 탈출 장치(`EscapeMachine`)가 모두 켜졌는지 검사하고, 조건을 만족하면 탈출 포탈(`EscapePortal`)을 소환하여 클리어 조건을 만듭니다.
* **라이프사이클 관리**: 게임의 일시정지(`Pause`, `Unpause`), 시스템 메모리 초기화(`Quit`) 등의 세부 동작을 수행합니다.

## 2. 보조 시스템

위 핵심 로직이 안정적으로 돌아갈 수 있도록 돕는 유틸리티 클래스들입니다.

### 2-1. CoroutineHelper (`CoroutineHelper.cs`)
유니티의 코루틴(Coroutine)은 기본적으로 `MonoBehaviour`를 상속받은 클래스에서만 실행할 수 있습니다. `CoroutineHelper`는 이를 극복하기 위해 전역에서 언제든 코루틴을 쉽게 실행할 수 있도록 도와주는 **싱글턴 헬퍼 클래스**입니다.
* `Game` 클래스처럼 `MonoBehaviour`를 상속받지 않은 곳이나 정적 함수 내부에서도 지연 처리(Delay)나 다음 프레임 콜백이 필요할 때 쓰입니다.
* **제공 기능**: `OnNextFrame()`(다음 프레임에 실행), `Delay()`(특정 시간 이후 실행), `Run()`(사용자 정의 IEnumerator 실행).

### 2-2. TimeManager (`TimeManager.cs`)
인게임 시간을 유니티 엔진의 전역 `Time.timeScale`에 전적으로 의존하지 않고 **독립적으로 관리**하기 위한 클래스입니다.
* 매 프레임 갱신될 때 자체적인 `TimeScale`을 `deltaTime`에 곱하여 고유의 `DeltaTime`을 계산합니다.
* **주요 쓰임새**: 게임 플레이 중 일시정지(`Pause`) 기능이 필요할 때, 유니티 자체의 `Time.timeScale = 0`을 사용하면 캔버스 UI 애니메이션과 각종 씬 이펙트까지 모두 멈춰버리는 문제가 발생할 수 있습니다. `TimeManager`를 쓰면 인게임 로직(캐릭터 이동, 시간 흐름 등)의 `TimeScale`만 0으로 만들어 **UI는 정상 작동하게 두고 게임 플레이만 멈추는 깔끔한 제어**가 가능해집니다.

### 2-3. Printer (`Printer.cs`)
유니티의 내장 `Debug.Log` 기능을 래핑(Wrapping)하여 만든 **전용 로그 출력 싱글턴 클래스**입니다.
* `Printer.Log()`, `Printer.LogWarning()`, `Printer.LogError()`를 제공합니다.
* **주요 쓰임새**: 클래스와 각 함수 위에 **`[Conditional("DEBUG")]`** 어트리뷰트가 적용되어 있습니다. 이를 통해 개발 빌드에서는 정상적으로 로그를 출력하면서도, 릴리즈(배포) 빌드로 컴파일될 때는 자동으로 로그 관련 코드가 삭제되어 **불필요한 로그 연산으로 인한 성능 저하를 방지**해 줍니다.
